Practical Observations

  • The current app is primarily built around the place ingestion stack, not the older unified items model.
  • public.places is the hottest table by sequential scans and is central to both public pages and admin flows.
  • public.items still exists and is sizeable, but appears more legacy and migration-oriented than active runtime data.
  • public.badges exists, but current "Kas Guide" badges are generated in code instead of being loaded from that table.

Recommended Next Steps
